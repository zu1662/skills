# 学习记录(Learning Record)格式

学习记录位于 `./learning-records/`,采用顺序编号:`0001-slug.md`、`0002-slug.md` 等等。目录按需创建——只在写第一条记录时建。

它们是教学的 ADR:捕获非显而易见的经验、关键洞察、以及被声明的先验知识——这些会引导后续会话。用于计算最近发展区。

## 模板

```md
# {学到的/确立的事的简短标题}

{1-3 句话:学到了什么(或确立了哪个先验知识),以及为什么对后续会话重要。}
```

就这样。整条学习记录可以是一段话。价值在于**记录下"现在已知此事"以及"它为什么改变了下一阶段该教什么"**——不在于填字段。

## 可选章节

仅当能带来真实价值时再写。大多数记录用不到。

- **Status** frontmatter(`active | superseded by LR-NNNN`)——当早期理解被证明是错的并被取代时很有用。
- **证据(Evidence)** ——用户如何展示了对该理解(答对了某题、完成了某练习、引用了过往经验)。当该主张未来可能需要重审时有用。
- **影响(Implications)** ——对未来会话,这解锁了什么或排除了什么。非常规影响时值得记录。

## 编号

扫描 `./learning-records/`,取最高编号 +1。

## 何时写一条学习记录

满足以下任一条件即可:

1. **用户对非平凡的概念展示了真正的理解** ——不只是接触,而是有证据能正确运用。这为下一步教什么设了新底线。
2. **用户披露了先验知识** ——"我已经会 X"。记下来,后续会话别再重教。同时记下用户声明的**深度**。
3. **某个误解被纠正** ——用户曾相信错误的认识,现在明白为何错了。这类记录价值很高:它们能预测未来在相关主题上的潜在卡点。
4. **使命在学习中发生转向** ——用户发现自己真正在乎的与最初想的不一样。交叉链接到 [MISSION.md](./MISSION-FORMAT.md) 并更新。

### 什么**不**算

- 只是"被讲到"的内容。讲到 ≠ 学到。等有证据再写。
- 已以术语定义形式压缩进 [GLOSSARY.md](./GLOSSARY-FORMAT.md) 的内容。别重复。
- 逐次会话的活动日志。学习记录不是日记——而是**决策级洞察**。

## 取代(Supersession)

当一条新记录与旧记录矛盾(用户的理解深化或被纠正),把旧记录标为 `Status: superseded by LR-NNNN`,**不要删除**。理解是如何演化的,这段历史本身就是有价值的信号。
