# GLOSSARY.md 格式

`GLOSSARY.md` 是本教学工作区的**标准语言**。所有讲解、练习、学习记录都应遵循其中的术语。**构建术语表本身就是学习的一部分**——把概念压缩成简洁定义,是用户真正理解的证据。

## 结构

```md
# {主题} 术语表

{一句话或两句话描述本术语表覆盖的主题。}

## 术语

**肌肥大(Hypertrophy)**:
由机械张力与代谢压力在反复训练中驱动的肌肉增长。
_避免用_:增肌、变大

**渐进超负荷(Progressive overload)**:
在时间维度上系统性地提升对肌肉的需求——通过负荷、容量或强度。
_避免用_:加把劲、升级

**RPE(自感用力程度,Rate of Perceived Exertion)**:
1–10 的自评打分,衡量一组练习主观感觉多难,10 表示力竭,8 表示还剩两次。
_避免用_:努力分数、强度评分
```

## 规则

- **只在用户理解了某个概念时再加术语。** 术语表是"压缩好的知识"记录,不是用户读来学习的字典。若用户刚被介绍这个概念,等到他们能正确运用后再晋升进术语表。
- **要敢取舍。** 同一概念有多个词时,挑最好的,把其余列为"避免用"。这就是语言压缩的机制。
- **定义要精炼。** 一两句话。定义"是什么",不是"做什么"或"怎么做"。
- **定义内部用术语表自己的词。** 一旦某个词进了术语表,在所有地方——包括其他定义里——优先用它。这就是让复杂概念后续更易理解的关键。
- **必要时用子标题分组**(如 `## 解剖`、`## 计划`)。当术语自洽时,扁平列表也行。
- **显式标记歧义。** 若某词在该领域内被宽松使用,记下本工作区的约定:"本工作区内,'组'始终指工作组数——热身单独追踪。"
- **随理解深化而修订。** 用户在第一周写的定义到第六周可能错了。就地更新,别留过期条目。
