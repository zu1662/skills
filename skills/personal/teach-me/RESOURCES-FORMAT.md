# RESOURCES.md 格式

`RESOURCES.md` 是本主题的可信资源精选。讲解所用的知识应从这里取,而非参数化猜测。智慧来自这里列出的社区。

## 结构

```md
# {主题} 资源

## 知识(Knowledge)

- [书:_力量训练的科学和实践_ — Zatsiorsky & Kraemer](https://example.com)
  关于训练计划与适应的奠基性著作。用于:周期化、恢复、强度区等任何相关问题。
- [文章:"我应该练多少?" — Greg Nuckols(Stronger By Science)](https://example.com)
  基于证据的训练量综述。用于:每周每肌群的组数目标。

## 智慧(Wisdom / 社区)

- [r/weightroom](https://reddit.com/r/weightroom)
  高信噪比 subreddit,有专人反 bro-science 治理。用于:计划评审、平台期排查。
- 线下:{健身房名} 周二力量训练课
  用于:动作的实时教练反馈。
```

## 规则

- **只要高可信度。** 优先一手资料、领域公认专家、同行评审工作、有强治理的社区。如果一个资源是营销伪装成教育,跳过。
- **每条都要注释。** 裸链接三个月后就毫无用处。加一行:它覆盖什么、何时拿出来。
- **按"知识/智慧"分组。** 与 [SKILL.md](./SKILL.md) 的哲学一致。允许某个资源只出现一组。
- **显式标注空缺。** 若使命所需的方向没有任何好资源,写 `## 空缺` 章节列出缺什么——这驱动后续搜索。
- **狠心修剪。** 后来被证明是错的、浅薄的、或偏离使命的资源,删掉而不是埋掉。五个锋利的资源胜过三十个平庸的。
- **记录社区偏好。** 若用户已表示不想加入社区,在这里注明,后续会话别再重复推荐。
