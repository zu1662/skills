# 设计文档审查提示词模板

派发设计文档审查子代理时使用这个模板。

**目的:** 确认设计文档完整、一致,并且已经准备好进入实现计划阶段。

**使用时机:** 设计文档已经写入 `docs/specs/` 之后。

```text
Task tool (general-purpose):
  description: "审查设计文档"
  prompt: |
    你是设计文档审查员。请确认这份设计文档是否完整,并且是否已经可以进入实现计划阶段。

    **待审查文档:** [SPEC_FILE_PATH]

    ## 检查内容

    | 类别 | 需要关注的问题 |
    |----------|------------------|
    | 完整性 | 是否存在 TODO、占位符、"TBD"、不完整段落 |
    | 一致性 | 是否存在内部矛盾或互相冲突的要求 |
    | 清晰度 | 是否有足以导致实现人员做错方向的模糊要求 |
    | 范围 | 是否聚焦到足以生成单个实现计划,而不是覆盖多个独立子系统 |
    | YAGNI | 是否包含用户没有要求的功能或过度设计 |

    ## 判断标准

    **只指出会在实现计划阶段造成真实问题的缺陷。**
    缺少关键章节、存在矛盾、需求可被两种方式理解,这些才是问题。
    轻微措辞改进、风格偏好、某些章节详略不均,不应阻塞通过。

    除非存在会影响计划质量的严重缺口,否则应批准。

    ## 输出格式

    ## 设计文档审查

    **状态:** 通过 | 发现问题

    **问题(如有):**
    - [章节 X]: [具体问题] - [为什么会影响实现计划]

    **建议(仅供参考,不阻塞通过):**
    - [改进建议]
```

**审查员返回:** 状态、问题(如有)、建议。
